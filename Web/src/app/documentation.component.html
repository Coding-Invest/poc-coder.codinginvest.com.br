Análise Detalhada da Solução

Esta documentação descreve uma aplicação de chatbot baseada em arquitetura de microsserviços, utilizando o protocolo MCP (Model Context Protocol) para integração com IA generativa.

Este projeto é uma prova de conceito (POC) que foi descontinuado devido a priorizações internas. Ele está sendo tornado público como uma contribuição para a comunidade do GitHub, exatamente como está.

Visão Geral

A arquitetura segue princípios de separação de responsabilidades, com projetos modulares em .NET 8.0 e Angular 20. A solução é dividida em quatro camadas principais: Web, Client, Server e Domain.

Web (Frontend Angular)

Definições e Conceitos

O projeto Web é uma aplicação Angular standalone, responsável pela interface de usuário (UI) do chatbot. Implementa um componente de chat em tempo real, utilizando Fetch API para comunicação streaming com o backend. O usuário envia prompts via formulário editável, e a resposta da IA é exibida incrementalmente.

Padrões Utilizados

  • Component-Based Architecture: Componentes standalone como ChatbotComponent e App.
  • Reactive Programming: Usando RxJS e signals para reatividade.
  • UI/UX Patterns: Chat minimizável com histórico dinâmico.
  • Build e Deployment: Angular CLI para gerar assets em wwwroot/browser.

Estrutura de Código

  • app.ts: Componente raiz com signal para título.
  • chatbot.component.ts: Lógica do chat com métodos como sendMessage().
  • chatbot.component.html: Template com diretivas Angular.

Client (API Gateway e Host Frontend)

Definições e Conceitos

O projeto Client é uma API ASP.NET Core que serve como gateway entre o frontend Angular e os serviços backend. Hospeda a aplicação Web e expõe endpoints para interação com IA via x.ai API e MCP.

Padrões Utilizados

  • API Gateway Pattern: Centraliza requisições e roteia para LLM ou MCP.
  • Mediator Pattern: ChatbotController coordena interações.
  • Streaming Response: Server-Sent Events simulados.
  • Dependency Injection: Registra IHttpClientFactory.

Estrutura de Código

  • Program.cs: Configuração de Kestrel e serviços.
  • Controllers/ChatbotController.cs: Endpoint POST /api/chatbot/send.
  • DataTransferObject/Request/: DTOs como Payload e CompletionRequest.

Server (Servidor MCP e Ferramentas)

Definições e Conceitos

O projeto Server implementa o servidor MCP, fornecendo ferramentas para automação de tarefas como leitura/escrita de arquivos e operações Git via JSON-RPC.

Padrões Utilizados

  • Command Pattern: Serviços como FileReadService.
  • Strategy Pattern: Switch-case no JsonRPCController.
  • JSON-RPC 2.0: Protocolo para comunicação.
  • Dependency Injection: Injeção de serviços.

Estrutura de Código

  • Controllers/JsonRPCController.cs: Roteia métodos para serviços.
  • Services/: Implementações como FileListService.
  • DataTransferObject/Request/: DTOs para cada ferramenta.

Domain (Biblioteca Compartilhada)

Definições e Conceitos

O projeto Domain define entidades compartilhadas como protocolos MCP e instruções para IA.

Padrões Utilizados

  • Shared Kernel Pattern: Entidades comuns.
  • Configuration Pattern: Tools com schemas.
  • Immutable Objects: Classes como ProtocolRequest.

Estrutura de Código

  • Tools.cs: Lista de ferramentas e schemas e prompt para IA.
  • ProtocolRequest.cs: Estrutura JSON-RPC.

Considerações Gerais

  • Arquitetura Geral: Clean Architecture com separação em camadas.
  • Integração: Projetos referenciam Domain; usa Newtonsoft.Json.
  • Segurança: CORS temporário, certificados autoassinados.
  • Pontos de Melhoria: Adicionar testes, logging e validação.
